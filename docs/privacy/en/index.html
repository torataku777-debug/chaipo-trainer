This Privacy Policy describes how Toratani Taku ("Developer") handles user information in the mobile application "OFC Neighbors" ("the App"). Please read this policy carefully before using the App.

1. Information We Collect

The App does not directly collect personally identifiable information. However, when using certain features (specifically, the friend-match feature), the App relies on Supabase as a backend, and the following information is stored on external servers.

1-A. Data stored only locally on your device

  • App settings (timer duration, initial score, other game settings)
  • Play logs and match history (AI matches, Fantasy Land practice)
  • Statistics related to Fantasy Land practice
  • Season start time
  • Other information necessary for game progression

This data never leaves your device.

1-B. Data stored on external servers (Supabase) when using friend-match

When using the friend-match feature (online play via 4-digit code sharing), the following information is stored on Supabase (a cloud service provided by Supabase Inc.) servers:

  • Anonymous user ID (auto-generated by Supabase, a non-identifying UUID)
  • Auto-generated display name (e.g., "Player-426e73", user-modifiable)
  • Match data (4-digit room code, placed cards, game progress, scores, Fantasy Land state)
  • Match creation timestamp, completion timestamp, and status (waiting / playing / finished / abandoned)

This information is necessary to synchronize gameplay with your match opponent. The Developer does not use this information for commercial purposes.

2. Disclosure to Third Parties

The App does not sell or disclose any user information to third parties (other users, companies, organizations, or individuals). However, when using the friend-match feature, the following information is visible to the match opponent (someone who shares the 4-digit code):

  • Display name (e.g., "Player-426e73", user-modifiable)
  • In-game placements and scores (information necessary for game progression)

This information is not shared with anyone other than the match opponent.

3. Advertising, Analytics, and Crash Reporting

The App does not use any of the following services or SDKs:

  • Advertising SDKs (AdMob, etc.)
  • Analytics or behavioral tracking tools (Google Analytics, Mixpanel, etc.)
  • Crash reporting tools (Firebase Crashlytics, Sentry, etc.)
  • User tracking (IDFA, etc.)

4. Data Retention and Deletion

4-A. Deleting locally stored data

  • Use the data management feature in the App's "Settings" screen
  • Uninstalling the App will delete all related data

4-B. Deleting data stored on Supabase

Match data (matches table) and user records stored on Supabase will be manually deleted by the Developer upon explicit request from the user. To request deletion, please contact us using the contact information below.

Additionally, completed matches (status=finished/abandoned) are scheduled to be automatically deleted after a fixed period (currently planned for 90 days).

5. Children's Privacy

The App is provided for general audiences without age restrictions. Since the App does not collect personally identifiable information, children may use it without parental consent. However, when using the friend-match feature, please understand that display names and game data are shared with match opponents, and use the feature under parental guidance.

6. In-App Purchases

The App (as of v1.1) does not provide in-app purchase functionality. If in-app purchases are introduced in the future, this policy will be updated to clearly describe how purchase-related information is handled.

7. Third-Party Services Used

The App uses the following third-party services for some of its features:

  • Supabase (https://supabase.com/privacy): Provides backend (authentication and data synchronization) for the friend-match feature. See the link above for Supabase's own privacy policy.

8. Changes to This Privacy Policy

This policy may change without notice in response to changes in laws or service content. We will notify users in advance on this page and within the App when making important changes. The revised policy applies once published.
