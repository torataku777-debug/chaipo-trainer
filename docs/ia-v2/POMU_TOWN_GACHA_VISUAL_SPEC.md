作成: 2026-05-13

## 1. 目的とゴール

`pomu_town_tab.html` のワイヤーフレーム(白背景・破線枠・[Pack1 背景] プレースホルダ) を、
**ポケポケのリリースレベル感**を持った視覚仕上げ版に置き換える。

評価基準:
- 「リリースレベルだ」と感じるトーン
- ポケポケが持つ「控えめ・上質・思考のスポーツ感」
- ぽむ世界観(青空 + パイナップル + ステッカー線画)が壊れない
- 装飾要素のごちゃごちゃは禁止、焦点は1〜2個に絞る

## 3. レイアウト構成

ワイヤーからの変更点を中心に記述。**基本構造(サブタブ・パックリスト縦並び・履歴行・BottomNav)は維持**。

### 3.1 全体背景

- ステータスバーから BottomNav の上までを **空ブルーの縦グラデーション**で塗る
- `background: linear-gradient(180deg, #e6f4fb 0%, #c5e2f0 100%)`
- グラデ中段右上に **薄い白雲(SVG または PNG)を1〜2個**、装飾として配置。控えめに(opacity 0.6 程度)
- 雲は装飾なのでクリック不可、コンテンツを邪魔しないこと

### 3.2 タウンヘッダー

- 「ぽむタウン」タイトル左、通貨ピル(コイン・ジェム)右
- 通貨ピル: 黒線囲み→ **白カード+ソフト影**に変更
  - 背景: cloud-white
  - 影: `0 1px 3px rgba(26,31,44,0.06)`
  - パイナップル黄のコインアイコン / 緑寄りの宝石アイコン(SVGまたは絵文字相当)

### 3.3 サブタブ(セグメントコントロール)

- 現状の「黒枠 + 黒背景アクティブ」は強すぎ → **ライトトーンに**
- 非アクティブ: 透明背景 + ink-500 文字
- アクティブ: cloud-white 背景 + ink-900 文字 + わずかな影
- コンテナ全体は 14px 角丸の白半透明カードに包む(空背景に浮かぶ感じ)

### 3.4 ガチャ情報行

- 「無料パック残り: 1/3 ・ 次のリセット: 8時間44分後」
- ink-500 で控えめに。装飾は無し
- 「1/3」の数字部分のみ pomu-yellow でハイライト

### 3.5 パックカード(★ 視覚仕上げの主役)

3つのパックそれぞれの構造は同一。差別化は中身の画像とぽむ表情で行う。

```
┌─────────────────────────────────┐
│ ┌────────┐  パック名(18/800)       │
│ │ ぽむ +  │  パック説明(12/500)     │
│ │ 包み画像 │                       │
│ │ 縦長カード │  ステータス行           │
│ └────────┘                       │
│  [無料で開ける]   [100コイン]       │
└─────────────────────────────────┘
```

- カード地: cloud-white、上端だけ cloud-soft へのグラデで柔らかさ
- カード角丸 16px
- パック画像領域: 96 × 132 (現状80×110から拡大、リッチ感を出す)
- パック画像領域は **透明** にして、ぽむ+包みのSVG/イラストが「カード地から少しはみ出して浮いて見える」表現

### 3.6 履歴行

- 現状の「破線+グレー」は弱い → ライト青の半透明カードに変更
- 背景: rgba(255, 255, 255, 0.55)
- 角丸: 14px
- 影は無し(目立たせない)

### 3.7 BottomNav

- 維持。ただし背景を半透明白(rgba(255,255,255,0.92)) + 上端に薄影
- アクティブタブの強調は黒ではなく ink-900 + 上端 3px の pomu-yellow バー

## 4. ぽむパックの視覚仕様 ★中核

### 4.1 共通の構造

各パックは **「包み + その隣のぽむ」のペア構図**。

- 包みは「カラフルな巾着袋」または「リボン付き紙包み」
- 包みの中心に **パックを象徴するアイコン**を小さく配置(背景パック=山と雲、フレームパック=四角フレーム、アバターパック=ぽむの顔シルエット)
- ぽむは包みの **右下に少しはみ出る形**で配置(包みの方が主役、ぽむは「これだよ」とお披露目する補助役)

**重要**: ぽむのポーズと表情は `pomu_character_sheet.png` の既存7種から流用。新規ポーズを描かない。

### 4.2 3パックの差別化

| パック | 包みの色・柄 | 中央アイコン | ぽむのポーズ |
|---|---|---|---|
| 背景パック | 空ブルー無地 + 白雲柄 | 山・空・雲のシルエット | 上向きの満足顔(キャラシ6番、目を閉じた笑顔) |
| フレームパック | 渋クリーム + 細格子 | 角丸フレーム枠 | 座って真剣な正面(キャラシ5番、座りポーズ) |
| アバターパック | パイナップル黄 + 緑葉 | ぽむシルエット(白丸+パイナップル) | お茶目な右向き(キャラシ7番、右向きで目を細める) |

### 4.3 状態表現

#### 4.3.1 is-ready(無料パック準備完了)

- カード枠: 通常の影 + **金色グロー(ready-glow-soft)** が外周に薄く広がる
- 包みの **上に1〜2個の星マーク**(SVG、控えめなアニメで0.5秒周期に発光)
- ステータステキスト: pomu-yellow + 太字 + ●印
- 「無料で開ける」ボタン: pomu-yellow → pomu-orange の縦グラデ + 白文字

#### 4.3.2 locked(クールダウン中)

- 包みとぽむを **彩度50%・明度+5%** にデサチュレート
- 包みの中央に **半透明の白い時計マーク**(SVG)を重ねる
- ステータステキスト: ink-500 + ⌛アイコン + タイマー(JetBrains Mono 等の monospace)
- 「無料で開ける」ボタン: グレースケール + disabled

### 4.4 アニメーション(微細・控えめ)

- is-ready 時のみ:
  - 包み全体が **2秒周期で ±2deg の傾き**(揺れ)
  - 星マークが **0.8秒周期で透明度 0.5 → 1.0**(明滅)
- ホバー/タップ時(モバイル含む):
  - 包みが **5% 拡大 + 100ms で復帰**(物理感)
- locked 時はアニメ無し(静止)

### 4.5 描画方法

**Codex への指示**: パック画像は **SVG をインラインで生成** すること。外部画像ファイルは不要。
- ぽむ: 既存キャラシのスタイル(白い丸+黒線+頭にパイナップル+黒い目)を SVG で再現
- 包み: 楕円底 + ふくらんだ袋 + 上部の絞り口、リボンや結び目を装飾
- 各SVGはサイズ 96×132 にフィット、scale で調整可能に
- アクセシビリティ: `role="img"`, `aria-label="背景パック"`等を付与

## 6. 参考素材と意図

### 6.1 トーン参考(既存スクショ)

- `references/IMG_2200.PNG` — タイトル画面の青空+ぽむのトーン。これを **タウンタブの空気感**として引き継ぐ
- ただし IMG_2200 のグランジ看板やマンガ線画的な要素は **採用しない**。ハンドオフ確定方針通り CG レンダーに振る

### 6.2 演出参考(モノポリーGo スティッカーパック)

参考にする要素:
- 「パックがタップで揺れる」物理感
- 「開封時に光が漏れる」期待感
- 「中身が1枚ずつめくれる」報酬感
- 「レアリティで光・色が変わる」差別化

採用しない要素:
- 「複数パックを同時に並べてバラ撒く」過剰演出
- 「派手な3D回転 + 爆発エフェクト」(チャイポは控えめ路線)

### 6.3 キャラクター素材

- `references/pomu_character_sheet.png` — ぽむ7表情。SVG化する際の参考にする
- ぽむは **絶対に変更しない**。既存のスタイル(白丸+黒線+パイナップル頭)を SVG で再現するだけ
- キャラシのナンバリング(私の観察):
  1. 正面・きょとん(上段左端)
  2. 正面・少し下を向く(上段2番目)
  3. 左横向き・笑顔(上段3番目)
  4. 後ろ向き(上段右端)
  5. 座りポーズ(下段左端)
  6. 目を閉じた満足顔(下段中央)
  7. 右横向き・目を細める(下段右端)
