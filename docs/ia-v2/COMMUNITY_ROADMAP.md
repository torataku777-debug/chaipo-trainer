## 戦略フレーミング

チャイポネイバーズはニッチな OFC ポーカーアプリ。コミュニティ強化は**生命線**であり、競合する大手CGゲームに対する**差別化の核**でもある。

### 3層の構造で考える

| 層 | 役割 | 担当チャネル |
|---|---|---|
| **A. アプリ内ソーシャル** | 既存ユーザー間の繋がり強化 | フレンドタブ、対戦中演出、プロフィール |
| **B. アウトオブアプリ** | 外部での拡散・コミュニティ醸成 | Discord、X、Reddit(後で実装) |
| **C. コンテンツ/UGC** | プレイヤー自身がコンテンツ生成 | ハンドシェア、リプレイ、シェアボタン |

## チャイポ固有の差別化要素

**ハンドシェア機能**(C層)が他のゲームに無い独自の強み。

OFCポーカーコミュニティは「具体局面の検討」が文化として根付いている:
- 「上段JK+A、中段K8762、最後の3枚どう置く?」
- これがX/note/Discordで日常的に議論されている

この**ハンドシェア機能**をアプリ内に組み込めば、ポケポケやMarvel Snapといった CGゲームに真似できないチャイポ固有のコミュニティ装置になる。

## 実装フェーズ

### v1.1(リリース時、最低限)

| 機能 | 説明 | 実装重さ |
|---|---|---|
| **シェアボタン** | 対戦結果・段位昇格・FL達成時にX/LINE/リンクコピーへ即シェア | 軽 |
| **プリセットスタンプ** | 対戦中に使えるポジティブ限定スタンプ(GG / Nice / ありがとう など) | 軽 |
| **基本フレンド機能** | 既存維持(コード、リスト、対戦履歴) | 既存 |

### v1.2(リリース後1ヶ月)

| 機能 | 説明 | 実装重さ |
|---|---|---|
| **アクティビティフィード** | フレンドの直近成果(段位UP、FL達成、スクープ等)が流れる | 中 |
| **公開プロフィールページ** | 段位・戦績・所持タイトルが見えるShareable URL | 中 |
| **称号/タイトルシステム** | 達成バッジを獲得・装備、プロフィールに表示 | 中 |

### v1.3(リリース後2ヶ月)— 差別化フェーズ

| 機能 | 説明 | 実装重さ |
|---|---|---|
| **ハンドシェア機能** ★ | 対戦中の任意局面を切り取って「これどう置く?」とX/Discordへ。チャイポ固有の強み | 重 |

### v2.0(リリース後3〜6ヶ月)

| 機能 | 説明 |
|---|---|
| コミュニティチャレンジ | 全プレイヤーで1つの目標(例: 今週 AA-FL 1000回達成) |
| デイリーパズル | 「シノさんEV最大の置き方を当てる」教育コンテンツ。Phase α-1の資産活用 |

### v3.0(1年後、人口次第)

- クラン・チーム機能
- ボイス無しチャット
- プレイヤー作成トーナメント
- 公式観戦モード
- ボイス・開放型チャット → **NG**(モデレーション地獄、ポケポケが拒否した理由)

## IA への影響

新タブは追加しない(5タブ上限)。既存タブを拡張:

| タブ | 追加 |
|---|---|
| フレンドタブ | アクティビティフィード(v1.2)、シェアボタン拡張(v1.3) |
| その他タブ | 公開プロフィールリンク、Discord/Xリンク(v2.0) |
| 対戦中(ゲーム画面) | プリセットスタンプ(v1.1)、対戦終了後ハンドシェアボタン(v1.3) |
| 全画面共通 | シェアボタン(v1.1) |

## 危険信号(回避すべきパターン)

- **開放型チャット**: モデレーション必須、個人開発では不可能
- **クランの早期実装**: 人口が無いと過疎クランが大量発生 → 逆効果
- **プレイヤー作成トーナメント**: 運営負荷大、不正リスク
- **ボイスチャット**: 安全性リスク、絶対NG
- **誹謗中傷を許す煽りスタンプ**: ポジティブ限定で初期実装

## 後で実装する外部チャネル

- **公式 Discord サーバー**: コアユーザーが集まる場、開発者と直接交流
- **X(Twitter)アカウント**: ニュース・更新告知・コミュニティハイライト
- **Reddit**: 海外展開時に追加
- **YouTube クリエイタープログラム**: ハイライト動画の創作者を支援

これらは**リリース時には間に合わなくてOK**(コアユーザーが集まり始めてから着手)。
